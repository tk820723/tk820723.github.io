Runloop源码解析

发表于 2021-08-08 分类于 iOS开发

Runloop概述

结构

  • Runloop本身是一个Event Loop循环，内部是用do while来实现
  • runloop内部有许多modes，存放在内部mutableSet里面。Mode又包含source observer timer三种item(用来回调)，
    这些mode代表内部runloop事件回传实体，在runloop获取的时候，如果没有就会创建对应的mode。
  • 真实存在的mode种类分为defaultMode以及trackingMode。正常情况下 defaultMode就是默认的mode，在默认调用runloop run的时候，会创建defaultMode。主线程的runloop通常情况就在这个defaultMode下运行，只有在滑动的时候会切换为trackingMode。
  • 还有一种mode是commonMode。这不是一个Mode。每当你把item加入到commonMode中，

Runloop Run全解析

注意此解析忽略所有加锁和解锁

  1. 调用run函数后会进入一个dowhile循环，内部调用CFRunLoopRunSpecific，默认mode为default。
  2. Specific这个函数首先会为runloop创建mode(如果没有mode)，并且创建perRunData(用来记录此次循环的stop以及wakeup标记)
  3. 然后通知observer runloop即将进入循环(kCFRunLoopEntry)
  4. 调用__CFRunLoopRun进入循环
  5. 进入真正的循环后，先检查是否被标记stop，如果标记了直接返回stop
  6. 如果是主线程的runloop，并且当前mode被标记为common，获取主线程的port记为dispatchPort
  7. 然后获取一下目前线程的port，为了timer使用
  8. 然后根据参数设置一下runloop超时数据，用dispatch_source将时间与__CFRunLoopTimeout回调绑定
  9. 再次进入一个内部的dowhile循环，在dowhile循环中拿到目前的portSet，waitSet中可能包含queuePort(线程Port)，timerPort(Timer)，wakeUpPort(外部唤醒Port)，src_port(source1中的port)
  10. 通知Observer timer以及source即将触发
  11. 进行runloop mode中的block
  12. 调用source0事件，这个source0一般属于应用级别事件的回调，比如说UITouchEvent，source0不能唤醒runloop，一般由source1进行唤醒
  13. 如果刚才第六步获取到了dispatchPort(主线程的port)，则向此port发送mach_msg进入内核询问是否需要处理msg，如果需要跳转到18handle_msg，直接跳过睡眠。（注意这里的__CFRunLoopServiceMachPort调用并不是重复轮询，第五个时间参数传入的是0，所以只会询问主线程的port一次）
  14. 如果没有获取到dispatchPort的话，则通知observer即将进入睡眠
  15. 再次进入dowhile循环，调用__CFRunLoopServiceMachPort切换至内核态进行睡眠。注意：这里的__CFRunLoopServiceMachPort是一直循环调用内核态的mach_msg轮询，应为时间传参为TIMEOUT_INFINITY
  16. 如果从睡眠中新来，livePort会变成当前唤醒的port，先去检查是不是当前mode的线程port。如果是当前的线程，检查是否是timer唤醒，如果不是timer继续进入睡眠；如果不是当前线程，直接跳出内部循环
  17. 通知observer runloop已经醒来
  18. handle_msg标志睡眠结束
  19. 接下来判断唤醒的port，如果是当前mode线程queue或者是mode的timerport则处理timer事件
  20. 如果port是主线程port，调用__CFRUNLOOP_IS_SERVICING_THE_MAIN_DISPATCH_QUEUE__处理主线程的msg
  21. 其他情况，寻找是否存在source1可以被处理，如果找到了source1，那么就处理source1的事件
  22. 再处理一下block中的事件
  23. 结束整个runloop过程 进入下一个while循环

Hexo中接入评论系统

发表于 2019-04-24 更新于 2021-08-08 分类于 Hexo

探索评论系统

折腾了好久的个人网站又迎来了他的新的功能：评论。
以前就折腾过一次这个，不过接入的是多说的服务，很多年没维护之后，这次重启了这个网站却发现，原来很多的评论系统都已经凉了，包括多说。
重新接入一个评论系统吧，找了很久，看有人推荐gitment，据说是基于git仓库来进行储存评论，感觉不会挂，所以我就抱着试一试的心态接入了。接完了一直不能登陆，一登陆就返回主页，报一个miss_match_redirectURL的错误，仔细检查了很久，发现没有配错什么东西
回到了gitment仓库搜索issues才发现，原来gitment的原理是通过作者建立的一个application验证，来授予外部访问的授权，然后再转存到你自己的仓库中。本来没什么，但是麻烦就处在这个application是有服务器验证的，于是在很久没维护之后，这个验证已经挂掉了，想要通过验证，需要自己建立一个验证服务器，略觉麻烦，遂放弃。有兴趣的同学可以通过这个教程来接入：https://imsun.net/posts/gitment-introduction/ 我遇到的问题其实也有解决方案：https://github.com/imsun/gitment/issues/178

转而尝试去接入另外的评论系统：来必力

接入来必力

来必力是韩国的产品，有中文的版本，有免费版本可以使用。接入方法也非常简单，这里使用我的主题hueman来做例子，一步一步记录一下接入的过程。

第一步：注册来必力

来必力的官方网站：https://www.livere.com
在这个网站注册一个账号，并且使用邮箱来验证你的账号，注册就大功告成了

第二步：初始化

回到首页，点击安装按钮，然后在city版本下点击安装，然后输入你的网站的连接（如我这个网站：https://spacevoid.cn)，网站名称随便，选择个人，然后点击申请获取代码来到下面这个页面

第三步：安装

复制图中框住的文字（div的整个block），回到你hexo的目录，打开主题目录下的layout/comment/index.ejs（其他主题也有类似的东西）,直接把之前的复制的文字粘贴在最底部就大功告成了，然后部署你的hexo就可以了。

第四步：管理

回到来必力的网站，点击右上角的个人图标，然后点击管理，就可以管理各种各样的东西，包括删除评论，新评论的提醒，登陆方法调整（点亮/拖动那些icons）等等，很简单，这里就不赘述了，大家自己研究就好。

更好的NSLog

自从Xcode9之后，系统的消息里头加入了很多的其实并不需要的log，再加上各种第三方的log，整个log console看的非常的杂乱，想要看见自己的log很困难。

第一步，让我们来一一解决掉系统的log
系统的log有两种类型，一个是UI新出的main tread checker的log，以及系统输出的NSLog

  • 干掉main tread checker只需要在scheme编辑中选择Diagnostics tag，然后将main tread checker的勾选消除掉，就可以屏蔽掉此类log。
  • 系统输出NSLog这个比较复杂，因为没有入口可以将它关闭，只能通过关闭NSLog输出来间接关闭系统的输出。有些童鞋会问，那NSLog被关闭自己想要输出怎么办，没关系，这个后面会说，这里先说下如何关闭NSLog的输出。想要关闭NSLog，也是在scheme编辑中，在Arguments tag下，增加”OS_ACTIVITY_DT_MODE”的key，然后设置value为NO，就可以关闭所有的NSLog的输出（包括第三方和自己的）

关于如何编写Hexo博客总结（二）

发表于 2016-06-17 更新于 2021-08-08 分类于 Hexo

Markdown编辑器

上一片讲了一些hexo用的markdown语法，这一片主要介绍一下在mac上编写markdown的时候使用的一些工具。

Ulysses

根据实际体验，与其说 Ulysses 是个富有创造性功能的应用，倒不如称它为融合了各类 Markdown 编辑器特点的集合体。

Ulysses算是名气最响的markdown编辑器了，功能上比较强大，除了实时编写实时查看效果以外，还自带功能强大的文档文件管理器，UI非常像邮件APP，也符合了markdown的宗旨。基本可以直接导入你的md文件夹进行编辑，无需每次编辑的时候再去导入，省去了不少麻烦。不过功能强大也带来了一些麻烦，Ulysses会自动对你编辑的东西进行一个自动格式调整，比如会自动把你本地图片换成标识符放在文章最底部，对于一些Hexo本身的插件会有冲突，所以我没有用这个，如果对于格式不用插件的同学，强烈推荐这个软件，使用起来真的很简单。

Mou

Mou 的最大優點是針對中文有比較多的優化，比如對中文輸入時發生行高「漂移」這一 Mac 文本編輯軟件普遍存在的問題做了處理；標點自動配對也有相應完善；還有一個很甜的功能 —— 直行書寫。另外預覽模式也比較靈活。我認為目前唯一缺憾就是所謂的「打字機模式」了，也就是把編輯點固定在文檔窗口的中心點。

Mou算是名气仅次于Ulysses的编辑器，功能与Ulysses差别不大，但是比Ulysses干净。Mou支持直接打开并且保存md文件，在控制台直接hexo new一个文件之后，就可以用mou来直接编辑了，非常方便。

Typora

Typora和前两款编辑器最大的不同，在于他查看效果的方式并不是分屏查看，而是编辑一行之后，立刻应用效果。

markdown的编辑器一搜有很多，这里就不一一介绍了，读者可以挑选一款适合自己编写习惯的来使用。

Markdown语法

Markdown 是一种轻量级的「标记语言」，它的优点很多，目前也被越来越多的写作爱好者，撰稿者广泛使用。看到这里请不要被「标记」、「语言」所迷惑，Markdown 的语法十分简单。常用的标记符号也不超过十个，这种相对于更为复杂的 HTML 标记语言来说，Markdown 可谓是十分轻量的，学习成本也不需要太多，且一旦熟悉这种语法规则，会有一劳永逸的效果。

Markdown 语法的简要规则

标题


标题是每篇文章都需要也是最常用的格式，在 Markdown 中，如果一段文字被定义为标题，只要在这段文字前加 # 号即可。

列表

熟悉 HTML 的同学肯定知道有序列表与无序列表的区别，在 Markdown 下，列表的显示只需要在文字前加上 - 或 * 即可变为无序列表，有序列表则直接在文字前加1. 2. 3. 符号要和文字之间加上一个字符的空格。

引用

只需要在文本前加入 > 这种尖括号（大于号）即可
